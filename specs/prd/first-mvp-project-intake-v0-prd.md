# PRD - First MVP Project Intake V0

## Problema

Ainda falta um fluxo padronizado para transformar uma ideia de MVP em PRD, SPEC e sprint plan sem construir produto completo.

## Objetivo

Preparar a entrada do projeto `demo-simple-web-mvp` com escopo minimo, foco em docs e zero execucao live.

## Escopo V0

- rascunho de PRD;
- rascunho de SPEC tecnica;
- rascunho de sprint plan;
- nenhuma instalacao de dependencia;
- nenhum deploy;
- nenhum push.

## Fora de escopo

- produto real em producao;
- autenticacao, pagamento ou integracao externa;
- qualquer execucao live de Codex.

## Guardrails

- segredo nunca vai para frontend;
- regra critica nunca depende do bundle;
- o backend continua sendo a borda de decisao.
