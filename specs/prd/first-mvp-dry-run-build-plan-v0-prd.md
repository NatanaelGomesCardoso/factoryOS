# PRD - First MVP Dry-Run Build Plan V0

## Problema

O intake do primeiro MVP já existe, mas ainda falta converter esse intake em um plano de build auditável e econômico.

## Objetivo

Gerar um plano dry-run que organize backend e frontend, preserve o roteamento para capsule e mantenha bloqueados push, deploy e APIs pagas.

## Escopo

- ler um report de intake existente;
- gerar tasks planejadas;
- separar backend e frontend;
- registrar routing para capsule quando a tarefa for `docs_only` ou `code_small`;
- gravar report local.

## Fora de escopo

- build real;
- deploy;
- push;
- mutação do repo real.
