# PRD - First Project Pilot Runbook V0

## Problema

O FactoryOS já possui as peças operacionais, mas ainda não tinha um runbook único para iniciar o primeiro projeto piloto de forma repetível.

## Objetivo

Definir um fluxo prático, curto e auditável para sair da entrada do projeto até a entrega local controlada.

## Escopo

- entrada do projeto;
- PRD;
- SPEC;
- sprint plan;
- build plan;
- capsule canary;
- apply gate humano;
- workspace scaffold;
- evaluator;
- delivery package;
- Obsidian sync;
- report retention.

## Fora de escopo

- push;
- deploy;
- API paga;
- secrets;
- execução live automática.

## Resultado esperado

- um comando de dry-run que descreve o fluxo operacional;
- pontos claros de aprovação humana;
- linguagem operacional curta e sem ambiguidade.
