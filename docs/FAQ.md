# FAQ

## Preciso saber programar?

Para usar o painel e ler reports, não muito. Para contribuir com código, sim. A documentação separa passos simples de referência técnica.

## O FactoryOS faz deploy sozinho?

Não. Deploy é bloqueado por padrão. Qualquer deploy precisa de autorização explícita e revisão.

## Ele faz push sozinho?

Não. Push automático é bloqueado por padrão.

## Posso usar em projeto antigo?

Sim. Use Reversa para retomar o projeto com leitura e planejamento antes de alterar arquivos.

## O que é cápsula?

Cápsula é um workspace pequeno e controlado para dar ao Codex só o contexto necessário. Ela reduz custo e risco.

## O que é report?

Report é um arquivo local com evidência: comando, entrada, decisão, status e próximo passo.

## O que é Reversa?

Reversa é o fluxo para entender projeto antigo e gerar material de retomada.

## Quando usar Codex caro?

Use só quando Python local, templates, roteamento simples, Ollama/local LLM ou cápsula barata não forem suficientes. Revisões pesadas e mudanças amplas exigem justificativa e gates.

## A Ajuda executa Mermaid?

Não. Mermaid aparece como código legível no painel local para evitar JS externo e dependências novas.

## Onde fica a memória?

Memória operacional fica no repo: docs, specs e reports. Conhecimento durável pode ir para Obsidian quando for útil para retomada futura.
