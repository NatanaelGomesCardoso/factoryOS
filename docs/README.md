# Documentação do FactoryOS

Esta pasta é o manual completo do FactoryOS. O README do GitHub é curto; aqui ficam os detalhes para usar, entender, validar e contribuir. O painel local expõe estes arquivos no Help Center read-only em `/help`.

## Índice

- [Primeiros passos](GETTING_STARTED.md): tutorial para abrir e validar localmente.
- [Guia de uso](USER_GUIDE.md): como operar projetos, reports e revisões.
- [Arquitetura](ARCHITECTURE.md): módulos, camadas de memória e diagramas.
- [Comandos](COMMANDS.md): referência dos comandos reais.
- [Fluxos](WORKFLOWS.md): receitas operacionais.
- [Segurança](SECURITY.md): bloqueios, ameaças e revisão humana.
- [Reversa](REVERSA.md): retomada de projetos antigos.
- [Painel local](LOCAL_PANEL.md): rotas e Ajuda local.
- [Runs estendidas](EXTENDED_RUNS.md): política de 2h e gates.
- [Limpeza e release](CLEANUP_AND_RELEASE.md): preparar GitHub público.
- [Estratégia de release](release-packaging-strategy.md): backup e export limpo antes do GitHub.
- [Export público limpo](clean-public-v1-export.md): plano, criação e validação local.
- [Gate público final](final-public-repo-readiness-gate.md): revisão final antes de publicar.
- [Contribuição](CONTRIBUTING.md): padrões para colaborar.
- [FAQ](FAQ.md): respostas simples.
- [Diagramas](diagrams/README.md): índice dos Mermaid.

## Como ler

Se você é leigo, comece por `GETTING_STARTED.md` e `FAQ.md`.

Se você vai contribuir, leia `ARCHITECTURE.md`, `COMMANDS.md`, `SECURITY.md` e `CONTRIBUTING.md`.

Se você vai publicar o projeto no GitHub, leia `CLEANUP_AND_RELEASE.md` antes de qualquer push.
